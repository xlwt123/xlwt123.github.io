css 盒子模型

在 css 中，有两种盒模型，分别是：IE 盒模型 (border-box) 和 W3C 标准盒模型 (content-box)。

一个盒子由 content（内容），padding（填充），border（边框），margin（外边界）组成。

IE 盒模型和 W3C 标准盒模型的区别：

  1. W3C 标准盒模型中，css 属性中的 width，height 只包含盒子的 content 内容。不包含 padding 和 border。
  2. IE 盒模型中，css 属性中的 width，height 包含了盒子的 content + padding + border。

可以通过 css 属性：box-sizing 来设置盒模型，默认为 content-box。

在 ie 6，7，8 中，若 DOCTYPE 缺失，则会将盒子模型解释成 IE 盒模型。如果声明了 DOCTYPE 类型，则所有浏览器都会将盒子模型解释成 W3C 标准盒模型。

BFC 和 IFC

BFC 即块级格式化上下文。BFC 的特性有以下几点：

  1. BFC 是一个独立的容器，容器内的元素不会影响到容器外的元素，同时，容器外的元素也不会影响到容器内的元素。
  2. 在 BFC 中，元素（盒子）都是从顶端开始垂直的一个接一个的向下排列。两个相邻盒子之间的间距取决于 margin 属性。
  3. 在 BFC 中，两个相邻的块级盒子之间的垂直外边距 (margin) 会发生重叠。

开启 BFC 的几种方式：

  1. 根元素或包含根元素的元素。
  2. 浮动元素，float ≠ none。
  3. 绝对定位元素，position = fixed | absolute。
  4. display = inline-block | flex | inline-flex | table-cell | table-caption。
  5. overflow ≠ visible。

  • 外边距重叠

    在 BFC 中，相邻的两个块级盒子之间的垂直外边距会发生重叠。两个块级盒子之间的边距取决于两个盒子之间 margin 值的较大者。

    • 两个相邻的外边距都是 正数 时，折叠外边距是 两者中较大的值 。
    • 两个相邻的外边距都是 负数 时，折叠外边距是 两者中绝对值较大的值 。
    • 两个相邻的外边距是 一正一负 时，折叠外边距是 两者相加的和。
