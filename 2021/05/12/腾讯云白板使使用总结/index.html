腾讯云白板使使用总结

Posted on 2021-05-12

在工作中使用到了腾讯云的白板，在此文章中进行总结。

白板运行原理

简单的说，白板 sdk 创建了一个 canvas 和其他内容的载体。

在 canvas 上进行操作时，会通过腾讯云 im 通知同个房间内的所有用户去同步当前用户的状态。使用课件也是类似的：pdf word 这两种类型的文件将会被转码成一张张图片，使用这两种类型的文档时，会在文件载体 dom 下，插入当前页的图片；ppt 则会转码成 html 文件，使用时通过 iframe 的形式插入文件载体的 dom 下。

简单的画一下直线和箭头

监听鼠标的移动事件，使用 canvas 中的 lineTo 方法，绘制出线条在 canvas 上。当然，这样子画出的线会跟随鼠标的轨迹，并不是一条直线。想要画出直线，还需要做一些额外的操作。

画一条直线，需要知道起点和终点的位置。鼠标点击时的坐标即是起点，移动时的坐标既是终点。所以只需要在每次移动事件触发时，先擦除掉 canvas 上的线，再根据起终点重新绘制出一条线即可。同时，还需要将之前绘制过的直线记录下来，重新绘制一遍。

画箭头，只需通过箭头和直线之间的夹角，计算出箭头两端的坐标点即可。

不规则的线

画一条不规则的线，看上去很简单，监听鼠标的移动事件，通过 lineTo 方法直接画出来就是一条不规则的线了。但是从直线的绘制上，我们知道，必须将每条线的信息都记录下来，在必要时重新绘制一次。同时，需要通过 im 通知其他用户，在其他用户的白板上也将这条线绘制出来，不可能将鼠标移动的每一个点都发给其他用户吧。这个问题思考了很久，后来也是同事告诉说可以通过 “贝塞尔曲线” 去绘制不规则的线。目前还在研究中。

不同 dpr 下兼容问题

在用户反馈中，有过这样一个问题：白板突然等比例缩小了。当时并没有太多的注意这个问题，只是反馈给了腾讯云让他们修复。在后来探究白板的实现原理中，大概知道了这个问题的原因。

由于 canvas 绘制出的文字，图片，线条属于位图，所以在放大之后就会失真，在高 dpr 下同样如此。想要解决失真问题，就需要根据 dpr 去对 canvas 的大小进行缩放，再通过 css 设置 canvas 元素的大小为设备的独立像素，再通过 canvas 的 scale 方法进行放大既可。

白板出现这个问题的原因，大概是某个地方将 canvas 的 scale 进行了错误的修改所导致的。
