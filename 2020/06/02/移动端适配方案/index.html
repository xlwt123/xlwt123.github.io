移动端适配方案

Posted on 2020-06-02

对于前端开发者而言，对于移动端不同设备的适配，可以说是必须掌握的知识点之一，本文将对这部分知识点进行总结。

关键概念

视口

在桌面浏览器下，正常的视口宽度就是浏览器的窗口宽度，会随着浏览器窗口的伸缩而缩放。默认情况下HTML标签会占满整个视口。

布局视口

在移动端上，如果移动端的视口表现和桌面浏览器的表现一致的话，可以想象网站的内容将会变得非常的小，所以为了避免这种情况：

在手机上，视口与移动端浏览器的宽度不再相关联，而是完全独立的了。我们称它为布局视口—CSS布局会根据它来计算，并被它约束

只有一小部分显示在设备的可见区域内，整个页面的大小就是布局视口，这个值可以通过document.documentElement.clientHeight/clientWidth获取。在移动端设备上，布局视口默认为980px（大部分）。

视觉视口

视觉视口是用户正在看到的网站的区域，用户可以通过缩放网页来改变视觉视口，但布局视口并不会发生变化。

可以通过window.innerWidth/innerHeight来获取视觉视口大小。

理想视口

由于在移动端上，布局视口的默认宽度并不是一个理想的宽度，所以苹果公司引进了理想视口这个概念：

它是对设备来说最理想的布局视口尺寸。显示在理想视口中的网页拥有最理想的浏览和阅读的宽度，用户刚进入页面时不再需要缩放。

在Chrome浏览器的控制台中看到的手机尺寸就是理想视口的尺寸，当页面缩放比例为100%时，理想视口就等于视觉视口：

可以使用meta标签告知浏览器，布局视口的宽度应该与理想视口的宽度一致。

<meta name="viewport" content="width=device-width"></meta>

分辨率

关于分辨率的细致讲解，请查看分辨率详解。

设备像素

设备像素设是物理概念，指的是设备中使用的物理像素。比如iPhone 5的分辨率640 x 1136px。

CSS像素

CSS像素是一个抽像的单位，主要使用在浏览器上，用来精确度量Web页面上的内容。一般情况之下，CSS像素称为与设备无关的像素(device-independent pixel)，简称DIPs。CSS像素顾名思义就是我们写CSS时所用的像素。

设备像素比

设备像素比：Device Pixel Ratio（简称DPR），是设备像素和理想视口（设备独立像素）的比值。

从苹果推出视网膜屏之后，才有设备像素比的概念：iPhone 5的设备像素是640，而理想视口宽度为320，将2x2个像素当成1个像素使用。

在js中，可以通过window.devicePixelRatio来获取dpr的值。在css中，可以通过device-pixel-ratio来获取dpr的值。

解决方案

rem

rem的大小是相对html节点的font-size来做计算的。当指定一个元素的font-size为2rem时，这个元素的实际字体大小为根元素html字体大小的两倍。

可以通过js获取屏幕宽度，然后根据宽度计算出相对应的字体大小并设置根元素的font-size，也可以通过媒体查询更改根元素的font-size。

局限

文字使用rem单位时，可能会出现这些问题：

  • 可能会出现通过rem计算，最终呈现到页面上是 23.335px 这样的奇葩的字体大小，可能还会因此出现锯齿、模糊不清等问题；
  • 对于大屏希望一行显示的文字多一些，小屏显示的少一些，而不是一视同仁的全部显示同样多的文字。

1px问题：

在dpr大于1的设备上，css像素所写的1px，实际上是被多个物理像素渲染，这就会出现1px在有些屏幕上看起来很粗的现象。

如何解决这一问题呢：

  • border-image或background-image

    通过媒体查询判断不同dpr给定不同的背景图。缺点：圆角不好处理，需要单独准备图片。

  • 伪类+scaleY

    通过媒体查询判断不同的dpr对线条进行缩放:transfrom:scaleY(1/dpr)。

  • 设置viewport

    通过设置页面缩放，让css像素等于真正的物理像素。当设备dpr为2时，设置viewport中initial-scale=1/2，再将页面根字体放大dpr倍。由于整个页面使用rem单位，所以页面看上去整体样式并没有什么变化。此时设置css像素1px，就是物理像素的1px。

    缺点：其他地方若需要使用px为单位时，需要判断dpr进行等比例缩放。

vw

vw是css3中的新特性，1vw即视觉视口宽度的1/100，对于移动设备来说，比如iphone6+的375pxCSS像素宽度，1vw就等于3.75px，通过这个单位可以解决上面的依赖于脚本绑定根元素font-size的问题，在竖屏和横屏下面都有比较好的效果。

局限
  • 同上
  • 当视觉视口宽度过宽，使用vw会拉的太大。rem可以通过限制根元素font-size大小让内容不至于扩得过大。

小程序rpx

小程序中，规定屏幕宽度为750rpx。如在 iPhone6 上，屏幕宽度为375px，共有750个物理像素，则750rpx = 375px = 750物理像素，1rpx =0.5px = 1物理像素。

图片模糊问题

我们平时使用的图片大多数都属于位图（png、jpg），位图由一个个像素点构成的，每个像素都具有特定的位置和颜色值。理论上，位图的每个像素对应在屏幕上使用一个物理像素来渲染，才能达到最佳的显示效果。而在dpr > 1的屏幕上，位图的一个像素可能由多个物理像素来渲染，然而这些物理像素点并不能被准确的分配上对应位图像素的颜色，只能取近似值，所以相同的图片在dpr > 1的屏幕上就会模糊:

解决方案

为了保证图片质量，我们应该尽可能让一个屏幕像素来渲染一个图片像素，所以，针对不同DPR的屏幕，我们需要展示不同分辨率的图片。

如：在dpr=2的屏幕上展示两倍图(@2x)，在dpr=3的屏幕上展示三倍图(@3x)。

    使用媒体查询判断不同的dpr来加载不同的图片

  • 使用image-set

  • 使用srcset

适配iPhone X

iPhone X增加了顶部的刘海和底部的小黑条，为了保证页面的显示效果，必须将页面限制在安全范围之内。

解决方案

viewport-fit是专门为了适配iPhoneX而诞生的一个属性，它用于限制网页如何在安全区域内进行展示。

配合cen(),constant()css函数，告诉浏览器安全范围，constant在iOS < 11.2的版本中生效，env在iOS >= 11.2的版本中生效（注意顺序）。
