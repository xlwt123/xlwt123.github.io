localStorage和sessionStorage

localStorage和sessionStorage是HTML5提供的技术，使得浏览器能够以一种更直观的方式存储键值对数据。

相同点
  • 存储大小均为5M左右
  • 都存在同源策略限制
不同点

  • 存储时间：

    • localStorage存储的数据是永久性的，直到用户删除
    • sessionStorage存储的数据与页面会话时间有关，一旦窗口或标签页被关闭，则存储的数据也会消失

  • 访问权限：

    • localStorage只要在相同的协议、相同的主机名、相同的端口下，就能读取/修改到同一份数据
    • sessionStorage要求更加苛刻，需要在同一个窗口下才能读取/修改到同一份数据
API

cookie和session

cookie

cookie是服务器发送到用户浏览器并保存在本地的一小块数据，他会在浏览器下次向同一服务器再发起请求时被携带并发送到服务器上。通常，它用于告知服务端两个请求是否来自同一浏览器，如保持用户的登录状态。

session

Session是在无状态的HTTP协议下，服务端记录用户状态时用于标识具体用户的机制。它是在服务端保存的用来跟踪用户的状态的数据结构，可以保存在文件、数据库或者集群中。

cookie与Web Storage区别

  • 存储大小：web storage存储大小可达5M，cookie则为4k左右
  • 时间限制：cookie可以设置失效时间
  • 性能问题：cookie在浏览器的每次请求中都会携带，web storage 存储在浏览器
